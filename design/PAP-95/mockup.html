Unified Chat — PAP-95

Conference Room ↔ Task Comments · proposed design

One conversation language, two surfaces

The conference room and task/issue comments should feel like the same product. Today each surface owns half of the right design — so the proposal is to cross-pollinate, not rebuild.

Conference room already has the blue/agent bubble language, but a stripped-down plain-text input. Task comments already have the rich composer (markdown, @mentions, attach) and a metadata-row system, but human comments render as neutral grey bubbles and bot status reads like a comment.

Blue is liveness

A human comment, a sent chat message, an active run — all blue. Blue is reserved for "something or someone is live."

Agents speak in cards

Agent replies are neutral card bubbles with a tail on the left — clearly a voice in the conversation, distinct from the human's blue.

Status is metadata

"Worked 2m", "updated this task", status changes — quiet rows on a rail, never bubbles. Useful, scannable, out of the way.

One composer

The same rich input (markdown, @mentions, attach, slash) in both surfaces. Task adds an assignee/status footer; that's the only difference.
